    Installing The LDM

    LDM Installing The LDM

    Overview

    Installing LDM involves preparing the system, building the software, configuring runtime files, and creating the product queue.

    System Requirements

    Supported Platforms

    • Linux (Ubuntu, Debian, CentOS, Rocky, Alma, Fedora, etc.)
    • macOS (Intel and Apple Silicon)
    • Other POSIX-compliant systems

    Required Software

    • C compiler (gcc or clang)
    • make, autoconf, automake
    • Compression libraries (zlib, libpng)
    • Standard TCP/IP networking utilities

    Recommended Resources

    • 2+ CPU cores
    • 4 GB RAM minimum
    • SSD for product queue
    • Stable broadband network

    Preparing The System

    sudo useradd -m -s /bin/bash ldm
sudo chown -R ldm:ldm /home/ldm

    Downloading And Building

    tar -xzf ldm-<version>.tar.gz
cd ldm-<version>
./configure --prefix=/home/ldm/ldm
make
make install

    Initial Configuration Files

    File Purpose
    ldmd.conf Upstream hosts, feed requests, server settings
    pqact.conf Pattern-action rules
    registry.xml Site metadata and queue parameters

    Creating The Product Queue

    ldmadmin mkqueue
pqmon

    Starting The LDM Daemon

    ldmadmin start
ldmadmin status
ldmadmin stop

    Verifying Connectivity

    ldmping <hostname>
notifyme -v -h <hostname>

    Firewall And Network

    The LDM uses TCP port 388. Ensure outbound and inbound rules allow this port and DNS works correctly.

    Post-Installation Checklist

    • LDM user exists and owns installation directory.
    • Product queue created and sized appropriately.
    • ldmd.conf and pqact.conf contain valid entries.
    • LDM daemon starts without errors.
    • ldmping and notifyme succeed.
