    Logging, Monitoring, And Troubleshooting

    LDM Logging, Monitoring, And Troubleshooting

    Overview

    LDM uses syslog for logging and provides utilities for monitoring queue health, connectivity, and product flow.

    Logging with Syslog

    Typical log locations:

    /var/log/ldm.log
/var/log/messages
/var/log/syslog

    ldmadmin log

    Log Levels

    Level Meaning
    error Critical failures requiring immediate attention
    warn Non-fatal issues that may affect performance
    notice Significant but expected events
    info Routine operational messages
    debug Detailed diagnostic information

    Monitoring Tools

    pqmon

    pqmon

    Shows queue size, product count, oldest/newest timestamps, and insertion rate.

    ldmping

    ldmping <hostname>

    notifyme

    notifyme -v -h <hostname>

    Common Issues

    No Data Arriving

    • Incorrect request lines.
    • DNS resolution failure.
    • Firewall blocking port 388.
    • Upstream host offline.

    “Product Not in Queue”

    • Queue too small for data volume.
    • Downstream processing too slow.

    pqact Not Processing Data

    • Incorrect regex or feedtype.
    • Missing exec line in ldmd.conf.
    • File permission or decoder errors.

    Monitoring Best Practices

    • Use pqmon regularly to check queue health.
    • Use notifyme when adding or modifying feed requests.
    • Rotate logs and monitor disk space.
    • Document local changes to pqact.conf.
