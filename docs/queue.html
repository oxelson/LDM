    The LDM Product Queue

    LDM The LDM Product Queue

    Overview

    The product queue is a memory-mapped circular buffer that stores recently received data products and enables low-latency, event-driven processing.

    Architecture

      • Implemented as a file on disk mapped into memory.
      • Provides near-RAM performance and crash resilience. Fixed size; oldest products are overwritten when full.

    Queue Contents

    Each product includes metadata (feedtype, timestamp, size, ID) and payload.

    Products enter the queue via upstream LDM, pqinsert, or PIPE reinjection.

    Retention Time

    Retention depends on queue size, data volume, product size, and burstiness.

    Queue Size Approximate Retention Time
    1–2 GB Minutes to ~1 hour
    4–8 GB Several hours
    20+ GB Many hours to a day

    Managing The Queue

    ldmadmin mkqueue
ldmadmin delqueue
pqmon

    Tuning Guidelines

    • Increase size if downstream clients see “product not in queue”.
    • Use SSD storage for best performance.
    • Avoid very small (<1 GB) queues.
    • Avoid very large (>100 GB) queues unless justified.

    Integrity and Recovery

    If the queue becomes corrupted or after a crash, rebuild it:

    ldmadmin delqueue
ldmadmin mkqueue

    Downstream Access

    Downstream clients request products by feedtype and pattern. If a product has been overwritten, they will see “product not in queue”.
