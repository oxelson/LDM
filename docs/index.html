    LDM User Guide

    LDM

    Introduction

    The Local Data Manager (LDM) is a distributed, event-driven data distribution system for real-time environmental data. It is the core technology behind Unidata’s Internet Data Distribution (IDD) system.

    Key Characteristics

    • Event-driven architecture with low latency.
    • Distributed provider/consumer/relay design.
    • Configurable workflows for ingest, storage, and forwarding.
    • High-performance memory-mapped product queue.
    • Pattern-based routing using regular expressions.

    Typical Use Cases

    • Receiving real-time weather model output.
    • Sharing radar or satellite products.
    • Archiving environmental data streams.
    • Operating an IDD relay node.
    • Supporting low-latency research workflows.

    Core Components

    Component Purpose
    ldmd Main LDM daemon; manages connections and data flow
    pqact Executes actions (store, decode, forward) based on patterns
    pqinsert Inserts locally generated data into the queue
    pqmon Monitors product queue status
    ldmping / notifyme Connectivity and data visibility diagnostics
    Product Queue High-performance circular buffer storing recent data
